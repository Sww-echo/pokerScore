# PokerScore 数据库查看命令

适用数据库：

- `PostgreSQL`
- 数据库名：`pokerscore`

当前实际表名：

- `app_user`
- `room`
- `room_event`
- `room_member`
- `room_round`
- `settlement_snapshot`
- `transfer_batch`
- `transfer_record`

## 1. 进入数据库


如果使用 `pokerscore` 用户：

```bash
psql "postgresql://pokerscore:2560910707gsw@127.0.0.1:5432/pokerscore"
```

退出：

```sql
\q
```

## 2. 查看表和表结构

查看所有表：

```sql
\dt
```

只看 `public` schema：

```sql
\dt public.*
```

查看所有表名：

```sql
SELECT tablename
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY tablename;
```

查看核心表结构：

```sql
\d public.app_user
\d public.room
\d public.room_member
\d public.room_round
\d public.transfer_batch
\d public.transfer_record
\d public.settlement_snapshot
\d public.room_event
```

## 3. 查看最近数据

最近用户：

```sql
SELECT id, nickname, auth_type, device_id, wechat_open_id, status, created_at
FROM public.app_user
ORDER BY created_at DESC
LIMIT 20;
```

最近房间：

```sql
SELECT id, code, name, status, owner_user_id, current_round_no, version, created_at, updated_at
FROM public.room
ORDER BY created_at DESC
LIMIT 20;
```

最近房间成员：

```sql
SELECT id, room_id, user_id, nickname, score, seat_label, is_online, joined_at, last_active_at
FROM public.room_member
ORDER BY joined_at DESC
LIMIT 30;
```

最近局：

```sql
SELECT id, room_id, round_no, status, version, settled_at, created_at
FROM public.room_round
ORDER BY created_at DESC
LIMIT 20;
```

最近转分批次：

```sql
SELECT id, room_id, round_id, request_id, operator_member_id, item_count, total_score, created_at
FROM public.transfer_batch
ORDER BY created_at DESC
LIMIT 20;
```

最近转分记录：

```sql
SELECT id, room_id, round_id, from_member_id, to_member_id, score, created_at
FROM public.transfer_record
ORDER BY created_at DESC
LIMIT 30;
```

最近结算：

```sql
SELECT id, room_id, round_id, request_id, settled_by_member_id, settled_at, created_at
FROM public.settlement_snapshot
ORDER BY settled_at DESC NULLS LAST, created_at DESC
LIMIT 20;
```

最近事件流：

```sql
SELECT id, room_id, round_id, operator_member_id, event_type, request_id, created_at
FROM public.room_event
ORDER BY created_at DESC
LIMIT 50;
```

## 4. 按房间码排查

先看最近房间：

```sql
SELECT id, code, name, status, current_round_no, created_at
FROM public.room
ORDER BY created_at DESC;
```

假设房间码是 `123456`。

查询该房间基础信息：

```sql
SELECT id, code, name, status, owner_user_id, current_round_no, version, created_at, updated_at
FROM public.room
WHERE code = '123456';
```

查询该房间成员：

```sql
SELECT rm.id, rm.user_id, rm.nickname, rm.score, rm.seat_label, rm.is_online, rm.joined_at
FROM public.room_member rm
JOIN public.room r ON r.id = rm.room_id
WHERE r.code = '123456'
ORDER BY rm.joined_at ASC;
```

查询该房间局信息：

```sql
SELECT rr.id, rr.room_id, rr.round_no, rr.status, rr.version, rr.settled_at, rr.created_at
FROM public.room_round rr
JOIN public.room r ON r.id = rr.room_id
WHERE r.code = '123456'
ORDER BY rr.round_no DESC;
```

查询该房间转分批次：

```sql
SELECT tb.id, tb.request_id, tb.item_count, tb.total_score, tb.created_at, rm.nickname AS operator_nickname
FROM public.transfer_batch tb
JOIN public.room r ON r.id = tb.room_id
LEFT JOIN public.room_member rm ON rm.id = tb.operator_member_id
WHERE r.code = '123456'
ORDER BY tb.created_at DESC;
```

查询该房间转分记录：

```sql
SELECT tr.id, r.code, fm.nickname AS from_nickname, tm.nickname AS to_nickname, tr.score, tr.created_at
FROM public.transfer_record tr
JOIN public.room r ON r.id = tr.room_id
JOIN public.room_member fm ON fm.id = tr.from_member_id
JOIN public.room_member tm ON tm.id = tr.to_member_id
WHERE r.code = '123456'
ORDER BY tr.created_at DESC;
```

查询该房间结算：

```sql
SELECT ss.id, ss.request_id, ss.settled_at, ss.created_at
FROM public.settlement_snapshot ss
JOIN public.room r ON r.id = ss.room_id
WHERE r.code = '123456'
ORDER BY ss.created_at DESC;
```

查询该房间事件流：

```sql
SELECT re.event_type, re.request_id, re.created_at, re.payload_json
FROM public.room_event re
JOIN public.room r ON r.id = re.room_id
WHERE r.code = '123456'
ORDER BY re.created_at DESC;
```

## 5. 快速查看最近一个房间

先查最近一个房间：

```sql
SELECT id, code, name, status, current_round_no
FROM public.room
ORDER BY created_at DESC
LIMIT 1;
```

拿到房间码后，替换下面的 `123456`：

```sql
SELECT rm.id, rm.nickname, rm.score, rm.seat_label, rm.is_online
FROM public.room_member rm
JOIN public.room r ON r.id = rm.room_id
WHERE r.code = '123456'
ORDER BY rm.joined_at ASC;

SELECT tr.id, tr.from_member_id, tr.to_member_id, tr.score, tr.created_at
FROM public.transfer_record tr
JOIN public.room r ON r.id = tr.room_id
WHERE r.code = '123456'
ORDER BY tr.created_at DESC;

SELECT re.event_type, re.request_id, re.created_at
FROM public.room_event re
JOIN public.room r ON r.id = re.room_id
WHERE r.code = '123456'
ORDER BY re.created_at DESC;
```

## 6. 非交互单条命令

不进入 `psql`，直接在 shell 里查最近房间：

```bash
psql "postgresql://postgres:你的密码@127.0.0.1:5432/pokerscore" -c "SELECT id, code, name, status, created_at FROM public.room ORDER BY created_at DESC LIMIT 10;"
```

直接查某个房间成员：

```bash
psql "postgresql://postgres:你的密码@127.0.0.1:5432/pokerscore" -c "SELECT rm.id, rm.nickname, rm.score, rm.seat_label FROM public.room_member rm JOIN public.room r ON r.id = rm.room_id WHERE r.code = '123456' ORDER BY rm.joined_at ASC;"
```
