# agentsww.com 首次上线最短执行清单

目标：

- 前端 H5：`https://agentsww.com`
- 后端 API：`https://agentsww.com/api/v1`
- Socket.IO：`https://agentsww.com/socket.io/`
- 部署方式：`Nginx + PM2 + Node.js 24 + PostgreSQL`

适用场景：

- 第一次把当前项目部署到你自己的云服务器
- 只保留最短必走步骤
- 默认服务器系统是 `Ubuntu 22.04`

## 0. 上线前确认

先确认这 4 件事：

- 域名 `agentsww.com` 的 `A` 记录已经指向云服务器公网 IP
- 安全组已放行 `22`、`80`、`443`
- 服务器可以正常访问公网
- 你已经准备好仓库地址

## 1. 安装基础环境

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git nginx unzip build-essential nano postgresql postgresql-contrib certbot python3-certbot-nginx
curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
sudo apt install -y nodejs
sudo npm install -g pm2
node -v
npm -v
pm2 -v
```

如果还是没有 `nano`，也可以直接用：

```bash
vi /srv/pokerscore/current/.env.production
vi /srv/pokerscore/current/server/.env
```

## 2. 上传代码到服务器

```bash
sudo mkdir -p /srv/pokerscore
sudo chown -R $USER:$USER /srv/pokerscore
cd /srv/pokerscore
git clone 你的仓库地址 current
cd /srv/pokerscore/current
```

## 3. 安装项目依赖

```bash
cd /srv/pokerscore/current
npm install
cd /srv/pokerscore/current/server
npm install
```

## 4. 启动 PostgreSQL 并创建数据库

```bash
sudo systemctl enable postgresql
sudo systemctl start postgresql
sudo -u postgres psql
```

在 `psql` 里执行：

```sql
CREATE USER pokerscore WITH PASSWORD '替换成你的数据库密码';
CREATE DATABASE pokerscore OWNER pokerscore;
GRANT ALL PRIVILEGES ON DATABASE pokerscore TO pokerscore;
\q
```

## 5. 先按 HTTP 配环境变量

前端：

```bash
cp /srv/pokerscore/current/.env.production.example /srv/pokerscore/current/.env.production
nano /srv/pokerscore/current/.env.production
```

改成：

```env
VITE_APP_BASE_PATH=/poker/
VITE_API_BASE_URL=/poker/api/v1
VITE_WS_BASE_URL=http://agentsww.com
VITE_WS_PATH=/poker/socket.io
```

后端：

```bash
cp /srv/pokerscore/current/server/.env.production.example /srv/pokerscore/current/server/.env
nano /srv/pokerscore/current/server/.env
```

改成：

```env
PORT=3001
JWT_SECRET=替换成至少32位随机字符串
JWT_EXPIRES_IN=7d
APP_H5_BASE_URL=http://agentsww.com/poker
DATABASE_URL=postgresql://pokerscore:你的数据库密码@127.0.0.1:5432/pokerscore
REDIS_URL=redis://127.0.0.1:6379
ROOM_CODE_LENGTH=6
```

## 6. 初始化数据库并构建项目

```bash
psql "postgresql://pokerscore:你的数据库密码@127.0.0.1:5432/pokerscore" -f /srv/pokerscore/current/docs/sql/init-v1.sql
cd /srv/pokerscore/current
npm run server:prisma:generate
npm run build
cd /srv/pokerscore/current/server
npm run build
```

如果后端日志报：

```text
@prisma/client did not initialize yet. Please run "prisma generate"
```

说明 `Prisma Client` 还没生成成功。直接按这个顺序补救：

```bash
pm2 stop pokerscore-server
cd /srv/pokerscore/current/server
npm uninstall @prisma/client
cd /srv/pokerscore/current/server
npx prisma generate --schema ../prisma/schema.prisma
npm run build
pm2 restart pokerscore-server --update-env
```

然后验证：

```bash
ss -lntp | grep :3001
curl -i http://127.0.0.1:3001/api/v1/health
```

原因说明：

- 当前项目的 `schema.prisma` 在仓库根目录 `prisma/`
- `prisma generate` 默认会把客户端生成到根目录的 `node_modules/@prisma/client`
- 如果 `server/node_modules` 里也单独装了一份 `@prisma/client`，TypeScript 和运行时会优先读到这份旧包，导致类型缺失或启动失败

## 7. 启动后端服务

```bash
cd /srv/pokerscore/current/server
pm2 start dist/main.js --name pokerscore-server --cwd /srv/pokerscore/current/server
pm2 save
pm2 startup
pm2 status
```

如果你之前已经误把 `ecosystem.config.local.cjs` 当脚本启动过，先清掉错误进程：

```bash
pm2 delete pokerscore-server.ecosystem.config.local
```

## 8. 配置 Nginx

```bash
sudo cp /srv/pokerscore/current/deploy/nginx/pokerscore.conf.example /etc/nginx/sites-available/pokerscore.conf
sudo ln -sf /etc/nginx/sites-available/pokerscore.conf /etc/nginx/sites-enabled/pokerscore.conf
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl enable nginx
sudo systemctl restart nginx
```

如果 `restart` 失败，再看：

```bash
sudo journalctl -u nginx -n 50 --no-pager
ss -lntp | grep :80
```

如果你的服务器装了宝塔 / aaPanel，需要额外注意：

- 真正生效的站点配置通常不在 `/etc/nginx/sites-enabled/`
- 而是在 `/www/server/panel/vhost/nginx/你的域名.conf`
- 实际运行的 `nginx` 也可能不是系统自带的 `/usr/sbin/nginx`
- 因此 `nginx -t` 通过，不代表宝塔那套站点配置也已经生效
- 同一个域名只能保留一份生效配置，否则会出现请求落到旧站点、被重定向到 `/en` 这类问题

这种情况下，优先检查：

```bash
which nginx
nginx -V 2>&1 | head -n 5
ps -ef | grep nginx
ls -lah /www/server/panel/vhost/nginx
sudo grep -RniE "server_name .*agentsww.com|NEXT_LOCALE|/en|proxy_pass" /www/server/panel/vhost/nginx 2>/dev/null
```

如果你要保留根站点，只新增 `https://agentsww.com/poker/` 给 PokerScore，不要覆盖整份站点配置，直接把下面片段加进
`/www/server/panel/vhost/nginx/agentsww.com.conf` 的 `server {}` 里：

```nginx
location = /poker {
    return 301 /poker/;
}

location ^~ /poker/api/v1/ {
    proxy_pass http://127.0.0.1:3001/api/v1/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

location ^~ /poker/socket.io/ {
    proxy_pass http://127.0.0.1:3001/socket.io/;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 600s;
}

location ^~ /poker/ {
    alias /srv/pokerscore/current/dist/;
    index index.html;
    try_files $uri $uri/ /poker/index.html;
}
```

改完以后执行：

```bash
sudo nginx -t
sudo systemctl reload nginx
curl -i http://127.0.0.1:3001/api/v1/health
curl -i -H "Host: agentsww.com" http://127.0.0.1/poker/api/v1/health
```

## 9. 先验证 HTTP 是否打通

```bash
curl http://agentsww.com/api/v1/health
```

本机排查时，建议同时执行：

```bash
curl -I -H "Host: agentsww.com" http://127.0.0.1
curl -i -H "Host: agentsww.com" http://127.0.0.1/api/v1/health
curl -i http://127.0.0.1:3001/api/v1/health
ss -lntp | grep :3001
pm2 status
```

判断方式：

- `curl http://127.0.0.1` 返回默认 Nginx 页面不代表站点配置已经命中，因为它可能只是默认站点
- 要看 `Host: agentsww.com` 请求是否命中你的 `pokerscore.conf`
- 正常的健康接口应该返回 JSON，至少包含 `code: 0`、`message: ok`
- 如果 `3001` 端口返回类似 `/en/api/v1/health` 这种内容，说明这个端口跑的不是当前 PokerScore 后端

然后浏览器打开：

```text
http://agentsww.com
```

这里必须确认：

- 首页能打开
- 创建房间正常
- 加入房间正常
- 转分和实时同步正常

如果这里都不通，不要继续申请证书。

