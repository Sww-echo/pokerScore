# agentsww.com 首次上线最短执行清单

目标：

- 前端 H5：`https://agentsww.com`
- 后端 API：`https://agentsww.com/api/v1`
- Socket.IO：`https://agentsww.com/socket.io/`
- 部署方式：`Nginx + PM2 + Node.js 24 + PostgreSQL`

适用场景：

- 第一次把当前项目部署到你自己的云服务器
- 只保留最短必走步骤
- 默认服务器系统是 `Ubuntu 22.04`

## 0. 上线前确认

先确认这 4 件事：

- 域名 `agentsww.com` 的 `A` 记录已经指向云服务器公网 IP
- 安全组已放行 `22`、`80`、`443`
- 服务器可以正常访问公网
- 你已经准备好仓库地址

## 1. 安装基础环境

```bash
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl git nginx unzip build-essential postgresql postgresql-contrib certbot python3-certbot-nginx
curl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash -
sudo apt install -y nodejs
sudo npm install -g pm2
node -v
npm -v
pm2 -v
```

## 2. 上传代码到服务器

```bash
sudo mkdir -p /srv/pokerscore
sudo chown -R $USER:$USER /srv/pokerscore
cd /srv/pokerscore
git clone 你的仓库地址 current
cd /srv/pokerscore/current
```

## 3. 安装项目依赖

```bash
cd /srv/pokerscore/current
npm install
cd /srv/pokerscore/current/server
npm install
```

## 4. 启动 PostgreSQL 并创建数据库

```bash
sudo systemctl enable postgresql
sudo systemctl start postgresql
sudo -u postgres psql
```

在 `psql` 里执行：

```sql
CREATE USER pokerscore WITH PASSWORD '替换成你的数据库密码';
CREATE DATABASE pokerscore OWNER pokerscore;
GRANT ALL PRIVILEGES ON DATABASE pokerscore TO pokerscore;
\q
```

## 5. 先按 HTTP 配环境变量

前端：

```bash
cp /srv/pokerscore/current/.env.production.example /srv/pokerscore/current/.env.production
nano /srv/pokerscore/current/.env.production
```

改成：

```env
VITE_API_BASE_URL=/api/v1
VITE_WS_BASE_URL=http://agentsww.com
```

后端：

```bash
cp /srv/pokerscore/current/server/.env.production.example /srv/pokerscore/current/server/.env
nano /srv/pokerscore/current/server/.env
```

改成：

```env
PORT=3000
JWT_SECRET=替换成至少32位随机字符串
JWT_EXPIRES_IN=7d
APP_H5_BASE_URL=http://agentsww.com
DATABASE_URL=postgresql://pokerscore:你的数据库密码@127.0.0.1:5432/pokerscore
REDIS_URL=redis://127.0.0.1:6379
ROOM_CODE_LENGTH=6
```

## 6. 初始化数据库并构建项目

```bash
psql "postgresql://pokerscore:你的数据库密码@127.0.0.1:5432/pokerscore" -f /srv/pokerscore/current/docs/sql/init-v1.sql
cd /srv/pokerscore/current
npm run server:prisma:generate
npm run build
cd /srv/pokerscore/current/server
npm run build
```

## 7. 启动后端服务

```bash
cp /srv/pokerscore/current/deploy/pm2/pokerscore-server.ecosystem.config.cjs /srv/pokerscore/current/deploy/pm2/pokerscore-server.ecosystem.config.local.cjs
cd /srv/pokerscore/current/server
pm2 start /srv/pokerscore/current/deploy/pm2/pokerscore-server.ecosystem.config.local.cjs
pm2 save
pm2 startup
pm2 status
```

## 8. 配置 Nginx

```bash
sudo cp /srv/pokerscore/current/deploy/nginx/pokerscore.conf.example /etc/nginx/sites-available/pokerscore.conf
sudo ln -sf /etc/nginx/sites-available/pokerscore.conf /etc/nginx/sites-enabled/pokerscore.conf
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx
```

## 9. 先验证 HTTP 是否打通

```bash
curl http://agentsww.com/api/v1/health
```

然后浏览器打开：

```text
http://agentsww.com
```

这里必须确认：

- 首页能打开
- 创建房间正常
- 加入房间正常
- 转分和实时同步正常

如果这里都不通，不要继续申请证书。

## 10. 申请 HTTPS

```bash
sudo certbot --nginx -d agentsww.com
sudo certbot renew --dry-run
```

## 11. 证书成功后，把环境变量改回 HTTPS

前端：

```bash
nano /srv/pokerscore/current/.env.production
```

改成：

```env
VITE_API_BASE_URL=/api/v1
VITE_WS_BASE_URL=https://agentsww.com
```

后端：

```bash
nano /srv/pokerscore/current/server/.env
```

改成：

```env
APP_H5_BASE_URL=https://agentsww.com
```

## 12. 重新构建并重启

```bash
cd /srv/pokerscore/current
npm run build
cd /srv/pokerscore/current/server
pm2 restart pokerscore-server
sudo systemctl reload nginx
```

## 13. 最终验收

检查：

- `https://agentsww.com`
- `https://agentsww.com/api/v1/health`

再做一轮业务验收：

1. 创建房间
2. 新开无痕窗口加入同一房间
3. 执行一次转分
4. 看两端分数是否同步
5. 执行结算
6. 查看结算页面
7. 执行重开局

## 14. 出问题先看这几个

```bash
pm2 logs pokerscore-server
sudo tail -f /var/log/nginx/error.log
sudo tail -f /var/log/nginx/access.log
sudo systemctl status postgresql
sudo certbot certificates
ss -lntp | grep 3000
ss -lntp | grep 5432
```
